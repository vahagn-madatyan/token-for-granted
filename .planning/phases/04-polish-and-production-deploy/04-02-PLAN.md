    **CI/CD pipeline (per DEPL-06):**
    Create `.github/workflows/deploy.yml` with the following configuration:

    ```yaml
    name: Deploy to Cloudflare Workers

    on:
      push:
        branches: [main]

    jobs:
      deploy:
        runs-on: ubuntu-latest
        name: Deploy
        steps:
          - uses: actions/checkout@v4

          - name: Setup pnpm
            uses: pnpm/action-setup@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '22'
              cache: 'pnpm'

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Typecheck
            run: npx tsc --noEmit

          - name: Build
            run: pnpm build

          - name: Deploy to Cloudflare Workers
            uses: cloudflare/wrangler-action@v3
            with:
              apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
              accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
              command: deploy
    ```

    Key details:
    - Trigger: push to main only (no PRs, no other branches)
    - Steps: checkout -> pnpm setup -> node setup -> install -> typecheck -> build -> deploy
    - Uses `pnpm/action-setup@v4` which auto-detects pnpm version from packageManager field or latest
    - Uses `cloudflare/wrangler-action@v3` for deployment (handles wrangler install internally)
    - No test step per user decision (CI/CD: typecheck -> build -> deploy, no tests)
    - Build step runs `pnpm build` which does `vite build && tsc --noEmit` (typecheck runs twice but that is fine -- explicit typecheck step gives clearer CI failure messages)
